# Dataset — Data Dictionary (Schema Reference)

> **Dataset will be populated after IRB-approved data collection (June 2026).**
> **No participant data exists yet. This directory contains schema documentation only.**

> **Note on commit history:** An earlier commit (0ab7eab) carries a
> message referencing a dataset. That dataset was synthetic/placeholder
> and was fully removed (commit 074f85d). No participant data exists in
> this repository. Real data will be collected only after IRB approval
> (June 2026). See docs/data_provenance.md.

Real data will be generated by running:
```bash
python tools/research/export_anonymized_dataset.py --out data/
```
after IRB approval and completion of the three-round data collection.

For development and unit testing, synthetic fixture files are available at
`tests/fixtures/` — see `tests/fixtures/README.md` for the mandatory warning
before use.

## events_anonymized.csv

| Column | Type | Description |
|--------|------|-------------|
| `participant_code` | string | Stable pseudonym (P001–P0XX). Never maps back to real identity in public files. |
| `ts_offset_seconds` | float | Seconds elapsed since that participant's first event in the round (wall-clock removed). |
| `action_type` | string | Event type (e.g. `SESSION_START`, `FLAG_SUBMIT_RESULT`, `TCP_CONNECT`). |
| `challenge_id` | string | Challenge identifier (e.g. `red_ghost_login`). |
| `success` | int | 1 = successful outcome for this event, 0 = otherwise. |

## skill_scores.csv

| Column | Type | Description |
|--------|------|-------------|
| `participant_code` | string | Stable pseudonym. |
| `accuracy` | int | Accuracy dimension score (0–100). |
| `persistence` | int | Persistence dimension score (0–100). |
| `web_recon` | int | Web Recon dimension score (0–100). |
| `protocol` | int | Protocol dimension score (0–100). |
| `ssh_pivot` | int | SSH Pivot dimension score (0–100). |
| `blue_analysis` | int | Blue Analysis dimension score (0–100). |
| `time_efficiency` | int | Time Efficiency dimension score (0–100). |
| `overall_level` | string | Categorical level (Beginner / Developing / Intermediate / Advanced). |
| `condition` | string | `control` or `treatment` (assigned by block randomization). |

---

## survey_responses.csv

| Column | Type | Description |
|--------|------|-------------|
| `participant_code` | string | Stable pseudonym. |
| `usability` | int | Likert 1–5: System usability rating. |
| `challenge_quality` | int | Likert 1–5: Challenge design quality. |
| `recommendation_quality` | int | Likert 1–5: Perceived quality of AI recommendations. |
| `confidence_improvement` | int | Likert 1–5: Self-reported confidence change. |
| `favorite_part_redacted` | string | Open response — PII scrubbed. |
| `improvement_point_redacted` | string | Open response — PII scrubbed. |
| `comments_redacted` | string | Open response — PII scrubbed. |

---

## expert_ratings.csv

| Column | Type | Description |
|--------|------|-------------|
| `participant_code` | string | Stable pseudonym. |
| `rater_id` | string | Expert rater identifier (R1, R2, R3). |
| `round_no` | int | Round number (1, 2, or 3). |
| `dimension` | string | One of the 7 skill dimensions. |
| `score` | float | Expert-assigned score (0–100). |

## manifest.json

Generated by `tools/research/generate_manifest.py` after data collection.
Contains SHA-256 hashes of all public CSVs and DB statistics for reproducibility verification.
